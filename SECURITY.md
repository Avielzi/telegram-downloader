## Reporting a Vulnerability

We take the security of Telegram Downloader seriously. If you discover a security vulnerability, we appreciate your help in disclosing it to us in a responsible manner.

**Please DO NOT open a public GitHub issue for security vulnerabilities.**

Instead, please report it directly to us via email at [aviel.ai@example.com](mailto:aviel.ai@example.com).

When reporting, please include:
- A clear and detailed description of the vulnerability.
- Steps to reproduce the vulnerability.
- The version of Telegram Downloader affected.
- Any potential impact or exploit scenarios.
- Your contact information for follow-up.

We will acknowledge your report within 48 hours and will work with you to understand and address the issue. We will keep you informed about the progress of the fix and will notify you once the vulnerability has been resolved.

## Security Best Practices

To ensure the security of your Telegram Downloader installation and data, we recommend the following best practices:

- **Keep Software Updated**: Always use the latest version of Telegram Downloader and its dependencies (Python, Telethon, PyQt6).
- **Secure API Credentials**: Never share your Telegram API ID and Hash. Keep them confidential and do not commit them to public repositories.
- **Regular Backups**: Regularly back up your downloaded media and any important session files.
- **Antivirus Protection**: Ensure your system has up-to-date antivirus software.
- **Firewall**: Use a firewall to restrict unauthorized access to your system.
